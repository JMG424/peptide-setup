2.3 Set Up the Gateway

The gateway is OpenClaw's brain — it runs in the background on your Mac and manages all your agents. Claude Code configures it to be secure by default.

⌨ Paste into Claude Code
Configure OpenClaw's gateway for security:
1. Make sure the gateway binds to loopback only (localhost)
   so nothing is exposed to the internet

3. Start the gateway: openclaw gateway
4. Verify it's running with: openclaw status
5. Show me the status output

The gateway should only listen on 127.0.0.1 — never on
a public IP.

Phase 3

Connect Discord

~20 minutes
Everyone does this on their own Mac + in Discord

Each leader gets their own Discord server with channels for each agent. OpenClaw has Discord built in — we just need to create a bot and plug in the token.

3.1 Create Your Discord Server

Each leader creates their own private Discord server for their AI team.

✋ Do this in Discord (app or browser)
  • Open Discord
  • Click the + button on the left sidebar → "Create My Own" → "For me and my friends"
  • Name it: [YourName] AI Team
  • Create these channels: #security, #systems, #chief-of-staff, #team
3.2 Create Your Discord Bot

Each leader creates a bot that will be their AI agents' voice in Discord.

✋ Do this in your browser — Jason walks you through it
  • Go to discord.com/developers/applications
  • Click New Application → name it PeptideAI-[YourName] → click Create
  • Click Bot in the sidebar
  • Click Reset Token → Copy the token → save it in a note (you need this in the next step)
  • Scroll down to Privileged Gateway Intents:
  • Turn ON: Message Content Intent
  • Turn ON: Server Members Intent
  • Click Save Changes
  • Click OAuth2 in the sidebar → URL Generator
  • Check: bot
  • Under Bot Permissions check: Send Messages, Read Message History, Add Reactions
  • Copy the URL at the bottom → open it in a new tab → add the bot to YOUR server
📝 You need 3 things for the next step

Have these ready: (1) Your bot token from step 3.2, (2) Your Discord user ID (enable Developer Mode in Discord Settings → Advanced, then right-click your name → Copy User ID), (3) Your server ID (right-click your server name → Copy Server ID).

3.3 Connect OpenClaw to Discord

Claude Code plugs your bot token into OpenClaw. Replace the 3 placeholder values with your real ones.

⌨ Paste into Claude Code — replace the 3 placeholders

   openclaw config set channels.discord.token --ref-provider default --ref-source env --ref-id DISCORD_BOT_TOKEN
   (first export the token as an env variable)
2. Enable Discord:
   openclaw config set channels.discord.enabled true

   so the bot responds to all messages in channels
4. Set my user ID as the only allowed DM sender
5. Restart the gateway
6. Verify the bot comes online in my Discord server

Go to your Discord server and send a message to make sure the bot is alive.

✋ Do this in Discord
  • Go to any channel in your new server
  • Type: Hello, are you there?
  • Your bot should respond within a few seconds
  • If it doesn't respond, tell Jason — we'll debug with Claude Code

Phase 4

Create Your 3 AI Agents

~30 minutes
Everyone does this on their own Mac

Each leader gets 3 agents: a Security Adviser, a Systems Designer & Analyst, and a Chief of Staff. Each agent has its own personality and expertise defined in a SOUL.md file — this is how OpenClaw knows what each agent is.

4.1 Create the Security Adviser

Claude Code creates the agent, its workspace, and writes its personality/rules.

⌨ Paste into Claude Code
Create an OpenClaw agent called "security" for my peptide
B2B company.

1. Create workspace: mkdir -p ~/.openclaw/agents/security
2. Register it: openclaw agents add security --workspace ~/.openclaw/agents/security
3. Create a SOUL.md file in that workspace with this content:

# Security Adviser

## Identity
You are the Security Adviser for a peptide B2B company.
You protect the business.

## What You Do
- Review business decisions for security risks
- Flag compliance issues (FDA, HIPAA, biotech regulations)
- Assess vendor and partner proposals for red flags
- Advise on data protection for formulations, trade
  secrets, and customer information
- Threat-model new systems and integrations

## How You Communicate
- Be direct. Flag issues with severity: LOW / MEDIUM /
  HIGH / CRITICAL
- Explain risks in plain English, not jargon
- Always recommend a specific action, not just "be careful"
- If something is fine, say so — don't invent problems

## Rules
- Never approve sharing proprietary formulations externally
  without explicit owner sign-off
- Always flag any request involving customer data
- If you're unsure about a regulation, say so clearly

4. Verify the agent is registered: openclaw agents list

Claude Code installs Tailscale and connects your Mac. Replace YOUR_TAILSCALE_KEY with the key Jason gave you.

⌨ Paste into Claude Code — replace YOUR_TAILSCALE_KEY

1. brew install --cask tailscale
2. Open the Tailscale app
3. Authenticate: sudo tailscale up --authkey=YOUR_TAILSCALE_KEY

6.3 Keep Mac Running 24/7

Claude Code configures your Mac to never sleep so your agents are always available.

⌨ Paste into Claude Code
Configure this Mac to run 24/7 as an always-on agent node:
1. Disable sleep: sudo pmset -a sleep 0 displaysleep 0

   (use openclaw daemon or a launchd plist)
4. Verify the gateway is running: openclaw status
5. Show me confirmation that sleep is disabled and the
   gateway auto-starts
