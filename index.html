2.3 Set Up the Gateway

exact commands in order and show me the output of each:

Phase 6 • Optional

Tailscale — Optional Remote Access

~20 minutes
Optional — skip if you don't want Jason to be able to SSH into your Mac

Tailscale creates a private encrypted network between your Mac and Jason's Mac so he can SSH in to fix things if something breaks. No ports are opened to the public internet — only Jason's devices on your private tailnet can reach your Mac. This phase is optional. If you'd rather handle support over a live screenshare when needed, skip straight to Phase 7.

⚠️ Per-machine keys only — never share

Each leader gets their own unique Tailscale auth key, generated by Jason specifically for your Mac. Do not reuse the same key across machines — if one key leaks, every machine using it is compromised. Jason creates a separate single-use key for each leader in the Tailscale admin console.

🔒 Claude Code can't run sudo

The remaining steps in this phase use sudo, which Claude Code is not allowed to run for you. The two steps below are marked manual — you paste the commands into Terminal yourself and type your Mac password when prompted. This is intentional: you should always know when something on your Mac is asking for admin rights.

6.1 Jason: Create Tailscale Account

Jason creates one Tailscale account for the team and generates auth keys.

✋ Jason does this in a browser
  • Go to login.tailscale.com → create an account
  • Go to Settings → Keys → Generate auth key
  • Create 3 keys (one per Mac) — pre-approved: YES
  • Give each leader their key
6.2 Install Tailscale on Your Mac

Paste each line into Terminal one at a time. Replace YOUR_TAILSCALE_KEY with the unique key Jason sent you for this Mac.

✋ Paste each line into Terminal — type your Mac password when asked
  • brew install --cask tailscale — installs the Tailscale Mac app
  • sudo tailscale up --authkey=YOUR_TAILSCALE_KEY — joins your tailnet
  • tailscale status — copy the 100.x.x.x address it prints and send it to Jason in Discord DM
  • Enable Remote Login so Jason can SSH in: open System Settings → General → Sharing, turn on Remote Login, and under "Allow access for" select Only these users and add your own user account.
6.3 Keep Mac Awake (Optional)

If you want your agents to be reachable even when you're away from your Mac, disable sleep. Skip this step on a personal laptop — you can simply keep the Mac awake while you're using the agents, and let it sleep normally the rest of the time. The OpenClaw gateway will auto-start whenever the Mac wakes up, because openclaw onboard --install-daemon (Phase 2.1) already installed it as a background service.

✋ Only if you want your Mac to never sleep — paste each line into Terminal
  • sudo pmset -a sleep 0 — prevents system sleep
  • sudo pmset -a disksleep 0 — prevents disk sleep
  • pmset -g — shows your current settings so you can confirm
  • To undo later: sudo pmset -a sleep 1 disksleep 10
💡 Softer alternative

Don't want to permanently disable sleep? Run caffeinate -i -t 28800 in a Terminal window when you want to keep the Mac awake for 8 hours — it expires automatically. Or just close the Terminal window to let it sleep normally.

  • Jason knows your Tailscale 100.x.x.x address (if you sent it to him)
  • Remote Login enabled in System Settings (if you want Jason to SSH in)

Phase 8

Weekly Kaizen Loop

~5 minutes
Everyone does this on their own Mac

Every Friday at 9am, your Chief of Staff gets a Discord ping reminding it to review the week and suggest 3 improvements. This is how your AI team gets better over time — small, weekly, automatic. Takes 5 minutes to set up.

8.1 Create a Discord Webhook

A webhook is a URL that lets outside scripts post a message into a specific Discord channel. We use one to ping the Chief of Staff every Friday.

✋ Do this in Discord
  • Right-click the #chief-of-staff channel → Edit Channel
  • Click Integrations → Webhooks → New Webhook
  • Name it Kaizen Bot
  • Click Copy Webhook URL
  • Paste the URL into a note — you'll need it in the next step
8.2 Schedule the Friday Kaizen Ping

Claude Code creates a tiny script and a cron job that fires every Friday at 9am local time. Replace YOUR_WEBHOOK_URL with the URL you just copied from Discord.

⌨ Paste into Claude Code — replace YOUR_WEBHOOK_URL
Set up a weekly Kaizen loop for my Chief of Staff agent.

My Discord webhook URL: YOUR_WEBHOOK_URL

Please do all of this:

1. Create the scripts directory if it doesn't exist:
     mkdir -p ~/scripts

2. Write ~/scripts/kaizen-friday.sh with this exact
   content, and substitute my real webhook URL for
   YOUR_WEBHOOK_URL:

#!/bin/bash
curl -X POST "YOUR_WEBHOOK_URL" \
  -H "Content-Type: application/json" \
  -d '{"content":"🔄 **Weekly Kaizen Review**\n\nIt is Friday. Time to improve your team.\n\n1. Review this weeks daily notes\n2. What worked well?\n3. What was confusing or missing?\n4. Suggest 3 specific improvements\n\nReply in this channel with your 3 improvements."}'

3. Make the script executable:
     chmod +x ~/scripts/kaizen-friday.sh

4. Add it to my crontab to run every Friday at 9am
   local time — DO NOT overwrite my existing cron jobs:
     (crontab -l 2>/dev/null; echo "0 9 * * 5 $HOME/scripts/kaizen-friday.sh") | crontab -

5. Show me the full crontab so I can confirm it's
   scheduled:
     crontab -l

6. Fire the script once right now so I can verify the
   ping actually lands in my #chief-of-staff channel:
     bash ~/scripts/kaizen-friday.sh

Note: cron only runs while the Mac is awake. If your
Mac is asleep at 9am Friday, the ping fires the next
time the Mac wakes up that week.
8.3 Verify the Test Ping Landed

Check your Discord #chief-of-staff channel. You should see a "Weekly Kaizen Review" message that was just posted by the webhook.
